# HSWQ V1.9 → V1.92 – Detailed Improvements

This document explains all improvements from V1.9 (SSIM 0.59) to V1.92, based on the actual code diffs.

---

## 1. DualMonitor overflow protection

In V1.9 calibration, squaring large FP16 outputs could overflow, producing NaN/Inf and corrupting the sensitivity computation.

### V1.9 (L350-369)

```python
def update(self, input_tensor, output_tensor):
    with torch.no_grad():
        out_detached = output_tensor.detach().float()
        self.output_sum += out_detached.mean().item()
        self.output_sq_sum += (out_detached ** 2).mean().item()
        # ... channel_importance ...

def get_sensitivity(self):
    if self.count == 0: return 0.0
    mean = self.output_sum / self.count
    return (self.output_sq_sum / self.count) - mean ** 2
```

### V1.92 (L350-380)

```python
def update(self, input_tensor, output_tensor):
    with torch.no_grad():
        out_detached = output_tensor.detach().float()
        # NaN/Inf prevention: clamp to avoid overflow when squaring large FP16 outputs
        out_clamped = torch.clamp(out_detached, -65504.0, 65504.0)
        mean_val = out_clamped.mean().item()
        sq_mean_val = (out_clamped ** 2).mean().item()
        # NaN/Inf guard
        import math
        if math.isfinite(mean_val) and math.isfinite(sq_mean_val):
            self.output_sum += mean_val
            self.output_sq_sum += sq_mean_val
        else:
            pass  # Skip corrupted batch
        # ... channel_importance ...

def get_sensitivity(self):
    if self.count == 0: return 0.0
    mean = self.output_sum / self.count
    variance = (self.output_sq_sum / self.count) - mean ** 2
    import math
    return variance if math.isfinite(variance) else 0.0
```

### Changes

| Item | V1.9 | V1.92 |
|---|---|---|
| Output clamping | None | `torch.clamp(-65504, 65504)` |
| NaN/Inf check | None | Guarded with `math.isfinite()` |
| Corrupted batch handling | Added as-is (NaN contamination) | Skipped |
| `get_sensitivity()` return | Raw variance (may be NaN) | Falls back to 0.0 if not `isfinite` |

## 2. Introduction of Hard VETO layers

In V1.9, profile data was only used to compute `search_low`. In V1.92, we add a VETO mechanism that **forces layers with mathematically unrepresentable FP8 distributions into FP16**.

### V1.9 `derive_hswq_strategy()` return value

```python
return alpha, beta, get_dynamic_search_low
# → 3-tuple: (alpha, beta, search_low_fn)
```

### V1.92 `derive_hswq_strategy()` return value

```python
return alpha, beta, get_dynamic_search_low, hard_veto_layers
# → 4-tuple: (alpha, beta, search_low_fn, VETO layer set)
```

### VETO logic added in V1.92 (L458–484)

```python
hard_veto_layers = set()
if model_profile:
    for name, prof in model_profile.items():
        if isinstance(prof, dict):
            k = prof.get("kurtosis", 0)
            m = prof.get("abs_max", 0)
            o = prof.get("outlier_ratio", 0)
            is_extreme_divergence = (o > 40)   # divergence
            is_extreme_kurtosis = (k > 20)     # kurtosis
            is_huge_magnitude = (m > 20)       # magnitude
            if is_extreme_divergence or is_extreme_kurtosis or is_huge_magnitude:
                hard_veto_layers.add(layer_base_name)
```

### Rationale for VETO thresholds

| Threshold | Meaning | Typical examples |
|---|---|---|
| `kurtosis > 20` | Extremely heavy-tailed / spiky distribution | `adaLN_modulation`, `t_embedder` |
| `outlier_ratio > 40` | abs_max/std > 40 (severe center/outlier divergence) | `feed_forward.w2` family |
| `abs_max > 20` | Values beyond the safe FP8 E4M3 range | e.g. `layers.28.adaLN_modulation.0` |

## 3. Profile key normalization (prefix stripping)

In V1.9, profile keys still contained prefixes. When `get_dynamic_search_low()` inside `derive_hswq_strategy()` looked up profiles, module names (`layers.0.attention.qkv`) did not match profile keys (`model.diffusion_model.layers.0.attention.qkv.weight`), so the profile data was effectively unused.

### V1.9 (L376–387)

```python
def derive_hswq_strategy(model_profile):
    # --- Purely mathematical search_low computation ---
    def get_dynamic_search_low(name, weight_tensor):
        profile_key = name + ".weight"
        prof = model_profile.get(profile_key, model_profile.get(name, {}))
        # model_profile keys are like \"model.diffusion_model.layers.0...\" so they do not match
```

### V1.92 (L387–411)

```python
def derive_hswq_strategy(model_profile):
    # [CRITICAL FIX] Automatically detect and strip model prefixes from profile keys
    if model_profile:
        sample_key = next(iter(model_profile))
        profile_prefix = ""
        for pfx in ZIT_PREFIXES:
            if pfx and sample_key.startswith(pfx):
                profile_prefix = pfx
                break
        if profile_prefix:
            normalized_profile = {}
            for key, val in model_profile.items():
                stripped_key = key[len(profile_prefix):] if key.startswith(profile_prefix) else key
                normalized_profile[stripped_key] = val
            model_profile = normalized_profile
            print(f"  [Profile Normalize] Stripped prefix '{profile_prefix}' ...")
    # From here on, use normalized model_profile
```

## 4. FP16 selection logic overhaul

In V1.9, FP16 layers were chosen solely by `DualMonitor.get_sensitivity()`. In V1.92, selection is done in two stages: **profile-based score + VETO exclusion**.

### V1.9 selection logic (L578–584)

```python
# Single metric: DualMonitor output variance only
layer_sensitivities = []
for name in target_modules:
    if name in dual_monitors:
        layer_sensitivities.append((name, dual_monitors[name].get_sensitivity()))
layer_sensitivities.sort(key=lambda x: x[1], reverse=True)
num_keep = int(len(layer_sensitivities) * args.keep_ratio)
keep_layers = set([x[0] for x in layer_sensitivities[:num_keep]])
# Problem: w2 layers dominate due to huge output scale, as variance explodes
# Result: other important layers (e.g. attention.out) are pushed to FP8, hurting SSIM
```

### V1.92 selection logic (L633–675)

```python
# Step 1: Double-normalize profile keys (also done on main side)
_norm_profile = {}
for _pk, _pv in model_profile.items():
    # Strip prefix + `.weight` suffix
    _stripped = _pk
    for _pfx in ZIT_PREFIXES:
        if _pfx and _stripped.startswith(_pfx):
            _stripped = _stripped[len(_pfx):]
            break
    if _stripped.endswith(".weight"):
        _stripped = _stripped[:-7]
    _norm_profile[_stripped] = _pv

# Step 2: Exclude VETO layers from the dynamic pool
layer_sensitivities = []
for name in target_modules:
        if name in hard_veto_layers:
        continue  # Exclude VETO layers from the pool
    prof = _norm_profile.get(name, {})
    k = prof.get("kurtosis", 0)
    o = prof.get("outlier_ratio", 0)
    m = prof.get("abs_max", 0)
    profile_score = k + o * 2.0 + m * 0.5   # composite score
    layer_sensitivities.append((name, profile_score))

# Step 3: Exclusive union of VETO + Dynamic
layer_sensitivities.sort(key=lambda x: x[1], reverse=True)
num_keep_dynamic = int(len(layer_sensitivities) * args.keep_ratio)
dynamic_keep_layers = set([x[0] for x in layer_sensitivities[:num_keep_dynamic]])
keep_layers = dynamic_keep_layers.union(hard_veto_layers)
# → VETO 34 layers + Dynamic 43 layers = up to 77 layers (no overlap)
```

### Selection logic comparison

| Item | V1.9 | V1.92 |
|---|---|---|
| Selection metric | `DualMonitor.get_sensitivity()` (output variance) | Profile score `k + o*2 + m*0.5` |
| VETO mechanism | None | Force FP16 when kurtosis>20, outlier>40, magnitude>20 |
| VETO vs Dynamic | — | VETO layers removed from Dynamic candidates (no overlap) |
| Dynamic pool | Top 25% of all 208 layers = 52 | Top 25% of 174 non-VETO layers = 43 |
| Final FP16 layer count | 52 | VETO 34 + Dynamic 43 = **77 layers** |
| w2 domination issue | Present (scale-dependent) | Resolved (w2 already FP16 via VETO) |

## 5. Parts that did NOT change

The following components are **identical** between V1.9 and V1.92:

- Model loading (`load_zit_model`)
- Calibration pipeline (`ZITCalibrationPipeline`)
- Tokenizer resolution (`resolve_tokenizer_offline`)
- HSWQ V4 optimizer call (bins=8192, candidates=1000)
- Continuous `search_low` computation (`get_dynamic_search_low`)
- Autonomous Alpha/Beta computation logic
- Output format (`comfy_quant`, `weight_scale`)
- SageAttention2 integration
